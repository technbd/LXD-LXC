## Incus:

**Incus** is an open-source platform for managing system containers and virtual machines (VMs). It is designed to provide a cloud-like experience for running Linux containers and VMs on a single server or across a cluster.

Incus is a community-driven **fork of LXD**, created and maintained by the Linux Containers community after Canonical took LXD in a different direction. It preserves the familiar LXD experience while being developed under an open governance model.


### Key Features:

#### 1. System Containers: 

Unlike Docker, which typically runs a single application per container, Incus runs full Linux system containers. Features include:
- Full init system (systemd) 
- Multiple services per container
- SSH access
- Persistent storage
- User and group management
- Package management (APT, DNF, etc.)
 

#### 2. Virtual Machines

Incus can also manage KVM-based virtual machines.


#### 3. Image Management

Incus supports downloading images from public and private image servers.



#### 4. Snapshots

Create snapshots before making changes.


#### 5. Live Migration

Move running containers or VMs between Incus servers.


#### 6. Storage Backends

Supported storage drivers include: ZFS, Btrfs, LVM, Ceph, CephFS, Ceph RBD, Directory storage

#### 7. Networking

Incus provides multiple networking options: Bridge, NAT, Macvlan, OVN, Physical NIC, passthrough, Routed networking


#### 8. Profiles

Profiles allow reusable configurations for: CPU limits, Memory limits, Storage, Networks, Devices


#### 9. Clustering

Incus supports clustering, allowing multiple servers to be managed as a single environment.


#### 10. Projects

Projects separate resources for different teams or environments. Like: Development, 
Production, Testing


#### 11. Web UI

Incus provides an official web interface for managing: Containers, Virtual machines, Networks, Storage. 



### Key Differences from LXD: 
| - | LXD | Incus | 
| - | --- | ----- | 
| Maintained by | Canonical | Linux Containers | 
| Install method | Snap | APT / native packages | 
| Command | `lxc` | `incus` | 
| Init command | `lxd init` | `incus admin init` | 
| License | Proprietary (Canonical) | Apache 2.0 |



### Incus vs Docker:

| Feature           | Incus | Docker                     |
| ----------------- | ----- | -------------------------- |
| System containers | ✅     | ❌                          |
| Full OS           | ✅     | ❌                          |
| Virtual machines  | ✅     | ❌                          |
| Runs systemd      | ✅     | Limited                    |
| Multiple services | ✅     | Typically one service      |
| Snapshots         | ✅     | Image-based workflow       |
| Live migration    | ✅     | No native support          |
| Clustering        | ✅     | Available via Docker Swarm |
| REST API          | ✅     | ✅                          |

### Install LXD via `apt`: 

_Install Incus on the control host:_
```
apt update
apt install -y incus incus-tools
apt install -y qemu-system-x86 qemu-utils
```


```
systemctl restart incus
```


```
incus version
```


### Post-install Setup:

Add user to the `incus` group:
```
sudo usermod -aG incus $USER
sudo usermod -aG incus-admin $USER
newgrp incus
```


### Initialize Incus:

```
incus admin init

### Run Containers and VMs with Incus

#### Launch a container:

```
incus launch images:ubuntu/24.04 my-container
```


```
incus list

+--------------+---------+---------------------+------+-----------+-----------+
|     NAME     |  STATE  |        IPV4         | IPV6 |   TYPE    | SNAPSHOTS |
+--------------+---------+---------------------+------+-----------+-----------+
| my-container | RUNNING | 10.49.19.121 (eth0) |      | CONTAINER | 0         |
+--------------+---------+---------------------+------+-----------+-----------+
```


```
incus config show my-container
```


_Assign CPU and Memory to an Existing Container:_
```
-- Limit to 2 CPUs: 
incus config set my-container limits.cpu 2


-- Limit Memory:
incus config set my-container limits.memory 512MiB


-- Verify Limits: 
incus config show my-container
```


_Access the container:_
```
incus exec my-container -- bash

Or,
incus shell my-container
```




#### Launch a VM:

```
incus launch images:ubuntu/24.04 my-vm --vm 
```


```
incus list

+--------------+---------+-----------------------+------+-----------------+-----------+
|     NAME     |  STATE  |         IPV4          | IPV6 |      TYPE       | SNAPSHOTS |
+--------------+---------+-----------------------+------+-----------------+-----------+
| my-container | RUNNING | 10.49.19.121 (eth0)   |      | CONTAINER       | 0         |
+--------------+---------+-----------------------+------+-----------------+-----------+
| my-vm        | RUNNING | 10.49.19.151 (enp5s0) |      | VIRTUAL-MACHINE | 0         |
+--------------+---------+-----------------------+------+-----------------+-----------+
```


```
incus config show my-vm
```


_Assign CPU and Memory to an Existing VM:_
```
-- Stop VM: 
incus stop my-vm


-- Limit to 2 CPUs: 
incus config set my-vm limits.cpu 2


-- Limit Memory:
incus config set my-vm limits.memory 512MiB


-- Verify Limits: 
incus config show my-vm
```


_Access the VM:_
```
incus exec my-vm -- bash

Or,
incus shell my-vm
```

## Accessing incus Containers/VM from External Users


### Option 1: Port Forwarding for Containers (Most Popular)


```
incus config device add my-container web-http proxy listen=tcp:0.0.0.0:8080 connect=tcp:127.0.0.1:80
```


```
incus config device show my-container

web-http:
  connect: tcp:127.0.0.1:80
  listen: tcp:0.0.0.0:8080
  type: proxy
```


```
incus config device remove my-container web-http
```


```
curl http://10.1.1.22:8080
```

_Add NIC with static IP to VM:_
```
incus config device add my-vm eth0 nic network=incusbr0 ipv4.address=10.49.19.151

incus restart my-vm
```


```
incus config device add my-vm web-http proxy listen=tcp:10.1.1.22:8080 connect=tcp:10.49.19.151:80 nat=true
```


```
incus config device show my-vm

  ipv4.address: 10.49.19.151
  network: incusbr0
  type: nic
web-http:
  connect: tcp:10.49.19.151:80
  listen: tcp:10.1.1.22:8080
  nat: "true"
  type: proxy
```


```
incus config device remove my-vm web-http
```


```
curl http://10.1.1.22:8080

### Useful commands:

```
-- List of configured remotes:
incus remote list


-- List available Ubuntu images:
incus image list images:
incus image list images: | grep ubuntu
incus image list images: | grep CONTAINER

incus image list images:ubuntu
incus image list images:ubuntu/24.04
incus image list images:ubuntu/noble


-- Show information:
incus info my-container


-- View profiles: 
incus profile list


-- Restart:
incus restart my-container


-- Delete:
incus delete my-container --force


-- Serial console (default)
incus console my-container


-- Take a snapshot:
incus snapshot my-container snap1


-- Restore a snapshot:
incus restore my-container snap1
```
